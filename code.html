Emulators in Python

Surrogate models, or emulators, are ways to mimic simulations that are otherwise too time intensive or computationally expensive to run directly. They have gained traction in the astrophysical literature for producing parameter constraints on sophisticated models of astrophysical processes. In Kern et al. 2017, we created a Python-based emulator framework for use in relating 21cm intensity mapping survey data to sophisticated models of Cosmic Dawn and the Epoch of Reionization.

We broke off the generalized emulator code for usage outside of parameter estimation problems, but also for usage outside of the context of 21cm cosmology. This code is called emupy.

For those interested in the code applied to 21cm cosmology, we wrote an external wrapper for handling data, constructing a likelihood function, importing emupy and training the emulator, and calling a Markov-Chain Monte Carlo sampler. This code is called pycape.

If you are interested in using the code(s), both are documented (to some degree), and have worked examples to get you started. If you use either codes in your research, please use the above citation.
